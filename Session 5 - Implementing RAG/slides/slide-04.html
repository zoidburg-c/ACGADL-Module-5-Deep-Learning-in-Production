The RAG Pipeline

User Question

→

Retriever

→

Relevant Documents

→

LLM + Context

→

Grounded Answer

Step 1

User asks a natural language question

Step 2

Retriever searches knowledge base for relevant docs

Step 3

Top matching documents are returned

Step 4

LLM receives question plus retrieved context

Step 5

Answer is grounded in actual source documents

The retriever bridges the gap between the user's question and the LLM's knowledge
