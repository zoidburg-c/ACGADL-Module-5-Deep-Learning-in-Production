# Ollama Setup Guide for Module 5: Deep Learning in Production

Ollama is a tool for running large language models (LLMs) locally on your own machine. It is used in **Session 4** (Deploying Models On-Premise) and **Session 5** (Implementing RAG) as a free, local alternative to cloud-hosted LLMs.

## System Requirements

| Requirement | Minimum | Recommended |
|-------------|---------|-------------|
| RAM | 4 GB | 8 GB or more |
| Disk space | 5 GB free | 10 GB free (for model downloads) |
| OS | macOS 12+, Linux (glibc 2.28+), Windows 10+ | macOS 13+, Ubuntu 22.04+, Windows 11 |
| GPU | Not required (CPU mode works) | NVIDIA GPU with 4+ GB VRAM for faster inference |

> **Note:** Ollama runs models on CPU by default. If you have an NVIDIA GPU, Ollama will automatically use it for faster inference. Apple Silicon Macs (M1/M2/M3/M4) use the built-in GPU automatically.

## 1. Installation

### macOS

**Option A: Download the app**

1. Visit [https://ollama.com](https://ollama.com)
2. Click **Download for macOS**
3. Open the downloaded `.dmg` file and drag Ollama to your Applications folder
4. Launch Ollama from Applications (it runs in the menu bar)

**Option B: Homebrew**

```bash
brew install ollama
```

After installation, start the Ollama service:

```bash
ollama serve
```

### Linux

Run the official install script:

```bash
curl -fsSL https://ollama.com/install.sh | sh
```

This installs Ollama and sets it up as a system service. The service starts automatically after installation.

To start it manually if needed:

```bash
sudo systemctl start ollama
```

### Windows

1. Visit [https://ollama.com](https://ollama.com)
2. Click **Download for Windows**
3. Run the installer (`OllamaSetup.exe`)
4. Follow the installation prompts
5. Ollama starts automatically and runs in the system tray

## 2. Pull the Required Model

Module 5 uses the **llama3.2:1b** model -- a small, fast model from Meta that is well-suited for learning and local development.

Open a terminal and run:

```bash
ollama pull llama3.2:1b
```

This downloads the model (~1.3 GB). The download may take a few minutes depending on your internet connection.

> **Why llama3.2:1b?** It is small enough to run on most machines (even without a GPU), fast enough for interactive exercises, and capable enough for the RAG and inference tasks in Sessions 4 and 5.

## 3. Verify Installation

### Check that the Ollama service is running

```bash
curl http://localhost:11434
```

Expected response:

```
Ollama is running
```

### Check that the model is downloaded

```bash
ollama list
```

Expected output (version numbers may vary):

```
NAME              ID            SIZE    MODIFIED
llama3.2:1b       <hash>        1.3 GB  <date>
```

### Test the model with a prompt

```bash
ollama run llama3.2:1b "What is machine learning in one sentence?"
```

You should see a response generated by the model within a few seconds.

### Test the API endpoint

The notebooks communicate with Ollama via its HTTP API. Verify it works:

```bash
curl http://localhost:11434/api/generate -d '{
  "model": "llama3.2:1b",
  "prompt": "Hello, how are you?",
  "stream": false
}'
```

You should receive a JSON response containing the model's reply.

## 4. Troubleshooting

### Ollama service not running

**Symptom:** `curl http://localhost:11434` returns "Connection refused"

**Solutions:**

- **macOS:** Launch the Ollama app from Applications. Check that the Ollama icon appears in the menu bar.
- **Linux:** Start the service with `sudo systemctl start ollama` or run `ollama serve` manually.
- **Windows:** Check the system tray for the Ollama icon. If not present, launch Ollama from the Start menu.

### Port 11434 already in use

**Symptom:** `ollama serve` fails with "address already in use"

**Solutions:**

1. Check what is using the port:
   ```bash
   # macOS / Linux
   lsof -i :11434

   # Windows (PowerShell)
   netstat -ano | findstr 11434
   ```

2. If another Ollama instance is already running, you do not need to start a new one.

3. If a different application is using port 11434, you can change the Ollama port:
   ```bash
   OLLAMA_HOST=0.0.0.0:11435 ollama serve
   ```
   Then update the notebooks to use `http://localhost:11435` instead.

### Model download fails or is very slow

**Symptom:** `ollama pull` hangs, times out, or reports network errors

**Solutions:**

- Check your internet connection
- Retry the pull command (it resumes from where it left off):
  ```bash
  ollama pull llama3.2:1b
  ```
- If behind a corporate proxy, set the proxy environment variables:
  ```bash
  export HTTP_PROXY=http://your-proxy:port
  export HTTPS_PROXY=http://your-proxy:port
  ollama pull llama3.2:1b
  ```

### Insufficient disk space

**Symptom:** Download fails with a disk space error

**Solutions:**

- The llama3.2:1b model requires approximately 1.3 GB of disk space
- Models are stored in:
  - **macOS:** `~/.ollama/models`
  - **Linux:** `/usr/share/ollama/.ollama/models` or `~/.ollama/models`
  - **Windows:** `C:\Users\<username>\.ollama\models`
- Free up space or change the model storage location:
  ```bash
  export OLLAMA_MODELS=/path/to/larger/drive/.ollama/models
  ```

### Slow inference (GPU vs CPU mode)

**Symptom:** Model responses take a very long time (30+ seconds)

**Solutions:**

- **Check if GPU is being used:** Run `ollama ps` while a model is loaded to see resource usage
- **NVIDIA GPU users:** Ensure NVIDIA drivers and CUDA are installed:
  ```bash
  nvidia-smi   # Should show your GPU
  ```
- **Apple Silicon users:** GPU is used automatically. No additional setup needed.
- **CPU-only machines:** Slower inference is expected. The llama3.2:1b model should still generate responses within 5-15 seconds on modern CPUs.
- **Reduce context length** if responses are very slow:
  ```bash
  ollama run llama3.2:1b --context-length 2048
  ```

### Model not found error in notebooks

**Symptom:** Notebook code returns "model not found" or similar error

**Solutions:**

- Verify the model is downloaded: `ollama list`
- Ensure the model name in the notebook matches exactly: `llama3.2:1b`
- Ensure Ollama is running: `curl http://localhost:11434`

## 5. Optional: Additional Models

The exercises only require `llama3.2:1b`, but you can experiment with other models if your machine has sufficient resources:

```bash
ollama pull llama3.2:3b        # Meta Llama 3.2 3B (~2 GB) - better quality
ollama pull phi3:mini           # Microsoft Phi-3 Mini (~2.3 GB)
ollama pull gemma2:2b           # Google Gemma 2 2B (~1.6 GB)
```

To remove a model and free disk space:

```bash
ollama rm <model-name>
```

## 6. Quick Reference

| Command | Description |
|---------|-------------|
| `ollama serve` | Start the Ollama service |
| `ollama pull <model>` | Download a model |
| `ollama list` | List downloaded models |
| `ollama run <model>` | Run a model interactively |
| `ollama ps` | Show running models |
| `ollama rm <model>` | Delete a downloaded model |
| `ollama --version` | Check Ollama version |
| `curl http://localhost:11434` | Check if the service is running |
